Flow Content

Flow content is the most common type of content in HTML documents. It includes elements that can be used to structure the main body of a document. Flow content can contain other flow content, as well as sectioning content, heading content, and more.

Examples of flow content include paragraphs, headings, lists, images, and more. It is the default type of content that you will use when creating web pages.

Flow content can be nested within other flow content, allowing for complex layouts and structures. It is essential for creating the main structure of a webpage, as it allows for the inclusion of text, images, and other media in a coherent manner.

What is Main/Aside?

    
Main is a semantic element that represents the main content of a document. It is used to encapsulate the
        primary content that is directly related to the central topic of the page. The main element should
        not be used more than once in a document.

    
Aside is a semantic element that represents content that is tangentially related to the main content of a
        document. It is often used for sidebars, pull quotes, or other content that provides additional context or
        information but is not essential to the main narrative.

What is Section/Article?

    
Section is a semantic element that represents a thematic grouping of content, typically with a heading. It is
        used to break up the content into logical sections, making it easier for users to navigate and understand the
        document structure.

    
Article is a semantic element that represents a self-contained piece of content that could be distributed
        independently from the rest of the document. It is often used for blog posts, news articles, or other
        content that can stand on its own.

What is Header/Footer?

    
Header is a semantic element that represents the introductory content or a set of navigational links for a
        document or section. It is typically used to contain the document's title, logo, and other relevant
        information.

    
Footer is a semantic element that represents the footer content of a document or section. It is typically used
        to contain information about the author, copyright details, and related documents or links.

What is Address?

    
Address is a semantic element that represents contact information for the author or owner of a document or
        section. It is typically used to provide details such as email addresses, phone numbers, and physical
        addresses.

What is Time/Mark?

    
Time is a semantic element that represents a specific time or date. It can be used to mark up dates, times, or
        durations in a machine-readable format.

    
Mark is a semantic element that represents text that has been highlighted or marked for reference or
        emphasis. It is typically used to indicate a portion of text that is of special interest or relevance.

What is Code/Pre/Kbd/Var/Samp/Cite/Dfn/Abbr/Sub/Sup/Small/Strong/Em/B/I/U/S/Del/Ins?

    
Code is a semantic element that represents a fragment of computer code. It is typically used to display code
        snippets or programming examples.

    
Pre is a semantic element that represents preformatted text. It is used to display text in a fixed-width
        font, preserving whitespace and line breaks.

    
Kbd is a semantic element that represents user input, typically keyboard input. It is used to indicate text
        that should be entered by the user.

    
Var is a semantic element that represents a variable in a mathematical expression or programming context. It is
        used to indicate a placeholder for a value that can change.

    
Samp is a semantic element that represents sample output from a program or command. It is used to display
        example output or results.

    
Cite is a semantic element that represents a reference to a creative work, such as a book, article, or
        website. It is typically used to provide citations or references in academic or scholarly contexts.

    
Dfn is a semantic element that represents a definition of a term. It is used to indicate the term being defined.
    

    
Abbr is a semantic element that represents an abbreviation or acronym. It is used to provide a full form or
        explanation of the abbreviation.

    
Sub is a semantic element that represents subscript text. It is used to display text that appears below the
        baseline, such as chemical formulas or mathematical expressions.

    
Sup is a semantic element that represents superscript text. It is used to display text that appears above the
        baseline, such as exponents or footnotes.
    

    
Small is a semantic element that represents text that should be displayed in a smaller font size, typically
        used for fine print or legal disclaimers.

    
Strong is a semantic element that represents strong importance or emphasis. It is typically displayed in bold
        text.

    
Em is a semantic element that represents emphasized text. It is typically displayed in italic text.

    
B is a semantic element that represents text that should be displayed in bold, but without conveying
        strong importance.

    
I is a semantic element that represents text that should be displayed in italic, but without conveying
        emphasis.

    
U is a semantic element that represents text that should be underlined, typically used for emphasis or
        highlighting.

    
S is a semantic element that represents text that has been struck through, typically used to indicate
        something that is no longer relevant or has been removed.

    
Del is a semantic element that represents text that has been deleted or removed from the document. It is
        typically displayed with a strikethrough style.

    
Ins is a semantic element that represents text that has been inserted into the document. It is typically
        displayed with an underline style.

    
These elements are essential for structuring and formatting text content in a meaningful way, allowing for
        better readability and understanding of the document's content.

     
        

        What is SVG/Canvas/Video/Audio/Details/Summary/Dialog?
    

    
SVG (Scalable Vector Graphics) is a semantic element that represents vector-based graphics. It is used to
        create scalable images that can be resized without losing quality. SVG elements can be used to create
        shapes, paths, and other graphical elements using XML syntax.

    
Canvas is a semantic element that represents a drawable region in HTML. It is used to create dynamic,
        scriptable rendering of 2D shapes and bitmap images. The canvas element provides a surface for drawing graphics
        using JavaScript.

    
Video is a semantic element that represents a video file. It is used to embed video content in a web page,
        allowing users to play, pause, and control video playback. The video element supports various video formats and
        provides built-in controls for user interaction.

    
Audio is a semantic element that represents an audio file. It is used to embed audio content in a web page,
        allowing users to play, pause, and control audio playback. The audio element supports various audio formats and
        provides built-in controls for user interaction.

    
Details is a semantic element that represents a disclosure widget from which the user can obtain additional
        information or controls. It is used to create expandable sections of content that can be toggled open or closed.
    

    
Summary is a semantic element that represents a summary or heading for the details element. It is used to
        provide a brief description of the content contained within the details element, allowing users to quickly
        understand the context before expanding the section.

    
Dialog is a semantic element that represents a dialog box or popup window. It is used to create modal or
        non-modal dialogs that can be used for user interaction, such as confirmation prompts or input forms. The dialog
        element provides a way to create interactive user interfaces without relying on external libraries or
        frameworks.
