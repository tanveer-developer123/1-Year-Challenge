Variants & State Separation

Visual interaction states are separated from base styling.
Variants help handle user interactions cleanly.
Keeps UI logic maintainable and scalable.

Interactive Button States

Tailwind Classes Explained: .bg-blue-500 /* Base background color */
.text-white /* Base text color */
.hover:bg-blue-700 /* On hover, background darkens */
.focus:outline-none /* Remove default outline on focus */
.focus:ring-2 .focus:ring-blue-300 /* Add blue ring on focus */
.active:bg-blue-800 /* On click, background is darkest */
.disabled:bg-gray-400 /* Disabled state: gray background */
.disabled:cursor-not-allowed /* Disabled state: not-allowed cursor */
Why use variants?
  • Variants keep interaction logic separate from base styling.
  • Easy to maintain and scale UI components.
  • Consistent user experience for all states.
