What Are Breakpoints?

Breakpoints are specific screen sizes where your design changes.

  • sm ≥ 640px
  • md ≥ 768px
  • lg ≥ 1024px
  • xl ≥ 1280px
  • 2xl ≥ 1536px
Tailwind Classes Explained: .grid /* Enables CSS Grid layout */
.grid-cols-1 /* 1 column on mobile */
.md:grid-cols-3 /* 3 columns at ≥768px */
.gap-6 /* 1.5rem gap between grid items */
.px-4 /* 1rem horizontal padding */
.sm:px-6 /* 1.5rem padding at ≥640px */
.md:px-8 /* 2rem padding at ≥768px */
.py-10 /* 2.5rem vertical padding */
How it works:
  • .grid-cols-1 makes a single column on mobile.
  • .md:grid-cols-3 switches to three columns at medium screens and up.
  • .gap-6 adds spacing between columns.
  • .px-4, .sm:px-6, .md:px-8 adjust horizontal padding at breakpoints.
  • This lets your layout adapt for phones, tablets, and desktops automatically.

Live Notes:

  • Resize to trigger different layouts.
  • Sidebar stacks below content on small screens.
  • Text adapts per breakpoint.
Sidebar Classes: .bg-blue-100 /* Light blue background */
.rounded-lg /* Large border radius */
.p-4 /* Padding on all sides */
.sm:p-6 /* More padding at ≥640px */
.md:p-8 /* Even more padding at ≥768px */
How it works:
  • Padding increases as screen size grows for better spacing.
  • Rounded corners and background color improve visual separation.
